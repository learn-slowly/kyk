## TL;DR
- **목표**: 권영국 후보의 정책과 비전을 효과적으로 전달하는 차세대 선거운동을 위한 고도화된 디지털 플랫폼 구축
- **기술스택**: Next.js 15 + React 19 + Sanity.io + Bootstrap 5 + Mustache + Vercel
- **핵심기능**: 실시간 정책 업데이트, 모바일 최적화 인터랙티브 UI, 캘린더 일정 관리 시스템, 비개발자 친화적 CMS
- **완성목표**: 2025년 5월 10일

## 1. 개요

본 문서는 권영국 대선후보의 공식 웹사이트 개발을 위한 요구사항을 정의합니다.

## 2. 목표

- 후보자의 정책과 비전을 명확하게 전달
- 선거 캠페인 정보와 일정 공유
- 디지털 유권자 접점 확대 (목표 유입 50만 MAU)
- 모바일 접근성 강화 (Lighthouse 점수 90+)
- 운영비 효율화 (월 인프라 비용 $50 이내)

## 3. 디자인 방향

### 3.1 색상 구성

- **기본 컬러 스키마**: 빨강-노랑-녹색 그라데이션
  - 빨강(#FF0000) → 노랑(#FFFF00) → 녹색(#00FF00)
- 이 그라데이션은 웹사이트 헤더, 버튼, 강조 요소에 활용
- Bootstrap 테마로 커스터마이징하여 일관된 디자인 유지

### 3.2 디자인 원칙

- 현대적이고 깨끗한 디자인
- 모바일 우선 반응형 레이아웃 (Bootstrap 그리드 시스템 활용)
- 접근성 준수 (WCAG 2.1 가이드라인 충족)
- 네비게이션 직관성 확보
- 인터랙티브 요소 활용 (Bootstrap + custom JS)

## 4. 주요 페이지

1. **히어로섹션**: 메인페이지. 권영국 후보 사진과 슬로건.
   - 풍부한 인터랙티브 요소와 애니메이션 적용
   - 그라데이션 배경과 호버 효과
   - 스크롤 트리거 기반 요소 등장 효과
   - 비주얼 중심의 단방향 정보 전달 (양방향 소통 없음)

2. **후보 소개**: 권영국 후보의 약력, 비전, 가치관
   - 인터랙티브 타임라인으로 경력 표시
   - 마우스 움직임에 반응하는 미디어 요소
   - 비주얼 중심의 스토리텔링 경험
   - 단방향 정보 전달 방식 유지

3. **정책**: 주요 정책 설명 (카테고리별 구분)
   - 정책 공약 핀터레스트 기능 (3D 회전 카드 인터랙션)
   - 드래그 가능한 정책 카테고리 필터
   - 스크롤 위치에 따른 애니메이션 효과
   - 정보 전달에 초점, 양방향 상호작용 제한

4. **뉴스 & 미디어**: 보도자료, 연설문, 언론 보도
   - 모바일 최적화 영상 플레이어 (Cloudinary 자동 화질 조절)
   - 관리자 친화적 CMS 인터페이스 구현
   - 컴퓨터 초보자도 쉽게 콘텐츠 업로드 가능
   - Sanity Studio 내 단순화된 워크플로우 및 가이드

5. **일정**: 선거 캠페인 일정, 유세 일정
   - 반응형 캘린더 시스템
   - 인터랙티브한 이벤트 표시 및 필터링
   - 일정 클릭 시 세부 정보 팝업
   - (구현 방식 추가 검토 중)

6. **참여하기**: 기부, 자원봉사, 지지 서명
   - 실시간 Q&A 피드 (Sanity GROQ 쿼리 기반)
   - 인터랙티브한 참여 양식 및 피드백
   - 사용자 상호작용 요소 적용 검토 중
   - (구현 방식 추가 검토 중)

7. **연락처**: 캠페인 본부 정보, 문의 양식
   - 인터랙티브 지도 및 위치 정보
   - 애니메이션 적용된 문의 양식
   - 시각적 피드백이 풍부한 UI
   - 단방향 정보 전달 중심 설계

## 5. 핵심 기능

- 소셜 미디어 공유
- 행사 알림 시스템
- 검색 기능
- 구글 애널리틱스, Plausible.io를 이용한 이용자 도달률 분석

## 11. 개인별 역할 및 책임

### 11.1 콘텐츠 담당자 역할

- **선거 공약 작성**: 후보의 주요 정책과 공약을 명확하고 설득력 있게 작성
- **후보자 소개 작성**: 권영국 후보의 이력, 성과, 비전을 설득력 있게 작성
- **뉴스 및 보도자료 작성**: 캠페인 관련 뉴스와 보도자료 작성
- **소셜 미디어 콘텐츠 제작**: 각 플랫폼에 맞는 콘텐츠 기획 및 작성
- **콘텐츠 업데이트**: 선거 일정에 맞춰 웹사이트 콘텐츠 정기적 업데이트
- **캠페인 일정 관리**: 캘린더 시스템을 통한 선거 일정 등록 및 업데이트

### 11.2 디자인 담당자 역할

- **웹사이트 UI/UX 디자인**: 사용자 경험을 고려한 전체 인터페이스 설계
- **로고 및 브랜딩 디자인**: 권영국 후보의 이미지와 메시지를 담은 로고 개발
- **그래픽 디자인**: 정책 인포그래픽, 배너, 포스터 등 시각 자료 제작
- **반응형 디자인**: 모바일, 태블릿, 데스크톱에 최적화된 디자인 구현
- **색상 스키마 적용**: 빨강-노랑-녹색 그라데이션을 효과적으로 활용한 디자인 요소 개발
- **캘린더 UI 디자인**: 직관적이고 사용하기 쉬운 캘린더 인터페이스 설계
- **Bootstrap 테마 커스터마이징**: 기본 Bootstrap 디자인을 캠페인 아이덴티티에 맞게 조정

### 11.3 개발 담당자 역할

- **프론트엔드 개발**: Next.js와 React를 활용한 웹사이트 구현
- **Bootstrap 통합**: Bootstrap 5 컴포넌트 및 그리드 시스템 적용
- **Mustache 템플릿 구현**: 동적 콘텐츠 렌더링을 위한 템플릿 시스템 구축
- **Sanity CMS 설정**: 콘텐츠 관리 시스템 설정 및 최적화
- **반응형 구현**: 디자인에 따른 반응형 웹사이트 구현
- **성능 최적화**: 웹사이트 로딩 속도 및 사용자 경험 최적화
- **배포 및 유지보수**: Vercel을 통한 웹사이트 배포 및 지속적인 기술 지원
- **캘린더 기능 개발**: react-big-calendar를 활용한 캠페인 일정 시스템 구현
- **API 연동**: 구글 캘린더 API 연동 및 데이터 동기화 구현

### 11.4 협업 프로세스

- 메신저를 이용한 실시간 협업
- Figma를 통한 디자인 공유 및 피드백
- Sanity 스튜디오를 통한 콘텐츠 관리 및 업데이트
